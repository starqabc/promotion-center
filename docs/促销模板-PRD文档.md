## 1. 整体说明

### 1.1 模块定位

促销模板是促销中心的**规则中枢**，用于预置各类促销活动的"能力开关"与"约束边界"。模板本身不直接面向消费者生效，而是作为**活动创建的蓝本**：业务人员基于已审核生效的模板快速创建促销活动，活动侧再填充具体商品、门店、时间等实例数据。

促销模板解决三个核心问题：
- **标准化**：统一管控"哪种促销类型允许做什么"（例如直降是否允许组合优惠、满减是否允许阶梯），避免活动配置随意化。
- **效率化**：将通用规则沉淀为模板，活动创建时一键继承，减少重复配置。
- **风控化**：通过模板状态机（草稿→审核中→有效→停用→作废）与引用校验，防止误操作影响线上活动。

### 1.2 业务边界

| 范围内 | 范围外 |
|--------|--------|
| 模板的增、删、改、查、复制 | 具体促销活动的创建与调度（在"促销活动"模块） |
| 模板类型与能力开关维护 | 商品/门店/会员等主档数据维护（在"主档信息"模块） |
| 模板审核与生命周期管理 | 模板生效后的算价与出券（在"促销计算"模块） |
| 模板被活动引用的校验 | 礼券主题与券包管理（在"礼券主题"模块） |

### 1.3 模板类型体系

模板分为 4 大类型，每类型下挂多个具体模板（共 18 个预置模板）：

| 模板类型（存储值） | 列表显示 | 含义 | 子模板示例 |
|--------------------|----------|------|------------|
| 直降 | 特价(直降) | 定价类，直接修改商品售价 | 一口价、最低免单、单品直降、满数量以上特价、组合特价 |
| 折扣 | 折扣 | 按比例打折 | 商品折扣、满金额折扣、满数量折扣、时段折扣、倍数折扣 |
| 满减满赠 | 满减满赠 | 达到门槛减现金或赠商品/权益 | 满金额减、满数量减、满金额赠、满数量赠 |
| 满赠券 | 满赠券 | 达到门槛赠送优惠券 | 满金额赠券、满数量赠券、满金额换购、满数量换购 |

### 1.4 核心概念

- **能力开关（Capability Switch）**：模板以开关形式声明该类型促销"允许"配置哪些规则。例如直降模板开启"直降/折扣"开关后，活动侧才能配置折扣值；未开启的能力在活动侧不可见。这是模板与活动的核心契约。
- **模板状态（status）**：模板生命周期标识，独立于"启用/禁用"开关。
- **启用状态（enableStatus）**：模板是否对外可用，与 status 正交。

```

        │
        ▼

        │
        ▼
```

## 4. 非功能需求（含权限设计方案）

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 列表查询响应 | 单次查询 ≤ 1s（模板总量预计 < 500 条） |
| 并发 | 支持 50 并发查询、10 并发写入 |
| 分页 | 默认每页 20 条，支持翻页 |

### 4.2 兼容性需求

- 浏览器：Chrome 90+、Edge 90+、Safari 14+。
- 分辨率：最低 1366×768，推荐 1920×1080。
- 终端：PC Web（H5 适配保留，但模板配置为 PC 主操作场景）。

### 4.3 数据安全与审计

- 所有模板变更（创建/编辑/提交/审核/停用/删除）需记录操作人、操作时间、操作前后快照，审计日志保留 ≥ 1 年。
- 模板编号生成需服务端统一分配，避免并发冲突。
- 敏感操作（删除、作废）需二次确认。

### 4.4 权限设计方案

模板模块采用**功能权限 + 数据权限 + 操作权限**三层模型：

#### 4.4.1 角色定义

| 角色 | 编码 | 说明 |
|------|------|------|
| 促销配置员 | `PROMO_CONFIG` | 创建/编辑/提交模板，不可审核 |
| 促销审核员 | `PROMO_AUDIT` | 审核/驳回模板，不可创建 |
| 促销管理员 | `PROMO_ADMIN` | 全部权限，含停用/作废/删除 |
| 只读查看者 | `PROMO_VIEW` | 仅查看列表与详情 |

#### 4.4.2 功能权限矩阵

| 操作 | 配置员 | 审核员 | 管理员 | 查看者 |
|------|:------:|:------:|:------:|:------:|
| 查询/查看列表 | ✅ | ✅ | ✅ | ✅ |
| 查看详情 | ✅ | ✅ | ✅ | ✅ |
| 新建/编辑/复制 | ✅ | ❌ | ✅ | ❌ |
| 保存草稿/提交审核 | ✅ | ❌ | ✅ | ❌ |
| 审核/驳回 | ❌ | ✅ | ✅ | ❌ |
| 停用/启用 | ❌ | ❌ | ✅ | ❌ |
| 作废/删除 | ❌ | ❌ | ✅ | ❌ |
| 导出/打印 | ✅ | ✅ | ✅ | ❌ |

#### 4.4.3 数据权限

- **按组织/门店隔离**：促销管理员可看全集团模板；区域配置员仅能维护本区域相关模板（通过 `object.storeOptions` 中的区域范围限定）。
- **创建人可见性**：配置员默认仅能编辑自己创建的草稿；管理员可编辑全部草稿。

#### 4.4.4 关键约束

- **审核分离**：同一人不可既提交又审核同一模板（防越权），提交人与审核人字段需不同。
- **状态校验优先**：所有操作先校验当前状态是否允许（见第 9 节状态规则），再校验权限。

## 6. 接口需求

### 6.1 模板分页查询接口

- **请求**：`POST /api/promo/template/query`
- **入参**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| pageNum | int | 是 | 页码，从 1 开始 |
| pageSize | int | 是 | 每页条数，默认 20 |
| templateNo | string | 否 | 模板编号，模糊匹配 |
| name | string | 否 | 模板名称，模糊匹配 |
| templateType | string | 否 | 模板类型：直降/折扣/满减满赠/满赠券 |
| status | string | 否 | 模板状态 |
| auditStatus | string | 否 | 审核状态 |
| timeType | string | 否 | 时间维度：create/audit/modify |
| timeFrom | datetime | 否 | 起始时间 |
| timeTo | datetime | 否 | 结束时间 |

- **出参**：`{ code, data: { total, list: [TemplateVO] } }`

### 6.2 模板保存接口

- **请求**：`POST /api/promo/template`（新建）/ `PUT /api/promo/template/{id}`（编辑）
- **入参**：`{ basic, object, conditions, rewards, display, targetStatus }`
  - `targetStatus`：`草稿`（保存草稿）或 `审核中`（提交审核）
- **校验**：
  - name 必填、type 必填；
  - 编辑时 status 必须为 `草稿`；
  - 提交审核时 name/type 不可为空。
- **出参**：`{ code, data: { id, templateNo, status } }`

### 6.3 模板状态流转接口

- 通用形态：`POST /api/promo/template/{id}/{action}`
- `action` 取值：`submit` / `approve` / `disable` / `enable` / `void`
- **停用接口特殊校验**：服务端需校验 `usedCount === 0`，否则返回业务错误码 `TEMPLATE_IN_USE`。

### 6.4 错误码约定

| 错误码 | 含义 | 触发场景 |
|--------|------|----------|
| TEMPLATE_NAME_REQUIRED | 模板名称必填 | 保存/提交校验 |
| TEMPLATE_TYPE_REQUIRED | 模板类型必选 | 保存/提交校验 |
| TEMPLATE_NOT_EDITABLE | 模板不可编辑 | 非 `草稿` 态编辑 |
| TEMPLATE_IN_USE | 模板已被引用，无法停用 | 停用时 usedCount > 0 |
| TEMPLATE_STATUS_INVALID | 状态流转非法 | 操作与当前状态不匹配 |
| TEMPLATE_NO_PERMISSION | 无操作权限 | 角色权限不足 |

## 8. 流程说明

### 8.1 新建模板流程

1. 用户在列表页点击"创建模板"，进入三步向导。
2. **Step1 基本策略**：填写模板名称、选择模板类型、填写说明、设置启用状态；促销对象区配置商品范围（固定开启）、费用承担开关。
3. **Step2 促销规则**：系统根据所选类型展示对应能力开关。用户勾选需要的能力：限量维度（整单/会员/门店/区域）、门槛维度（满金额/满数量/阶梯/倍数）、优惠维度（直降折扣/满减赠/赠券/上限）。
4. **Step3 预览&生成**：预览卡片实时展示配置摘要，底部点击"生成模板&提交审核"，模板进入 `审核中` 状态；或顶部"保存草稿"暂存为 `草稿`。

### 8.2 模板审核流程

1. 配置员提交模板（状态=审核中）。
2. 审核员在列表点击"审核"，校验配置合理性后通过。
3. 审核通过：状态=有效，系统自动在促销活动侧生成该模板名称的二级菜单。
4. 审核驳回：状态回退为草稿（需补充驳回原因，记入审计）。

### 8.3 模板停用作废流程

1. 管理员对 `有效` 模板点击"停用"。
2. 系统校验 `usedCount`：若 > 0，拦截并提示"已被活动使用，无法停用"。
3. 校验通过：状态=停用，已生效的活动不受影响（按快照运行）。
4. 停用态可"启用"回到有效，或"作废"进入终态。
5. 作废态可"删除"彻底移除。

### 8.4 异常流程

| 场景 | 处理 |
|------|------|
| 草稿未填名称直接提交 | 拦截，提示"请填写模板名称" |
| 非草稿态点击编辑 | 跳转详情页（只读） |
| 停用被引用的模板 | 拦截，提示"已被促销活动使用，无法停用" |
| 并发编辑同一草稿 | 后提交者提示"模板已被他人修改，请刷新" |
| 审核驳回 | 状态回退草稿，记录驳回原因 |

## 10. 操作解释

### 10.1 列表页操作

| 操作 | 入口 | 前置条件 | 执行结果 |
|------|------|----------|----------|
| 查询 | 查询按钮 | 无 | 按条件刷新列表 |
| 重置 | 重置按钮 | 无 | 清空筛选条件 |
| 创建模板 | 工具栏按钮 | 有新建权限 | 跳转新增向导 |
| 批量删除 | 工具栏按钮 | 已勾选 + 管理员权限 | 二次确认后删除勾选项 |
| 批量复制 | 工具栏按钮 | 已勾选 | 深拷贝生成新草稿，名称加" - 复制" |
| 导出 | 工具栏按钮 | 有导出权限 | 导出当前筛选/勾选的模板 |
| 编辑 | 行操作 | status=草稿 + 有权限 | 跳转编辑向导 |
| 提交 | 行操作 | status=草稿 | 状态 → 审核中 |
| 审核 | 行操作 | status=审核中 + 审核权限 | 状态 → 有效 |
| 详情 | 行操作 | 无 | 打开详情抽屉 |
| 停用 | 行操作 | status=有效 且 usedCount=0 + 管理员权限 | 状态 → 停用 |
| 启用 | 行操作 | status=停用 + 管理员权限 | 状态 → 有效 |
| 作废 | 行操作 | status=停用 + 管理员权限 | 状态 → 作废 |
| 删除 | 行操作 | status=作废 + 管理员权限 | 物理删除，二次确认 |

### 10.2 新增/编辑向导操作

| 操作 | 入口 | 执行结果 |
|------|------|----------|
| 下一步 | 步骤底部 | 校验当前步骤后进入下一步 |
| 上一步 | 步骤底部 | 回到上一步，已填数据保留 |
| 保存草稿 | 顶部按钮 | 校验 name/type 后保存为草稿，可继续编辑 |
| 生成模板&提交审核 | Step3 底部按钮 | 保存并提交，状态 → 审核中 |
| 返回列表 | 顶部按钮 | 离开向导（草稿已保存则保留） |

### 10.3 详情抽屉操作

| 操作 | 入口 | 执行结果 |
|------|------|----------|
| 切换步骤 | 步骤导航 | 切换只读详情步骤 |
| 基于模板新建活动 | 抽屉主按钮 | 关闭抽屉，进入活动创建并预填该模板 |
| 关闭 | 关闭按钮 | 关闭抽屉 |

### 10.4 能力开关交互规则

- **联动显示**：选择模板类型后，Step2 的能力开关按第 7.4 节矩阵显示/隐藏。
- **互斥规则**：
  - 满减赠优惠形式中，"商品"与"金额/比例"互斥，不可同时勾选。
  - 直降类型不支持"倍数"开关。
- **开关默认值**：单品优惠默认勾选；上限口径默认"数量"。
- **预览实时性**：Step2 任何开关变化，Step3 预览即时刷新。
