# 促销活动：满数量赠本品 / 满数量赠它品 与商品选择弹窗改造设计

## 1. 背景

本次仅调整 `促销活动` 模块，继续保持纯前端原型实现方式：

- 仅 HTML / CSS / JS
- 无依赖
- 可直接 `file://` 打开
- 复用现有菜单、弹窗、向导页、表格模式

当前已有：

- `满数量赠本品` 菜单已替代历史 `满数量赠`
- `满数量赠本品` 商品范围中，赠送方式已固定为本品
- 单组 + 倍数场景已放开参与范围多行

本次新增和修正的是：

- 新增同级菜单 `满数量赠它品`
- 满数量赠本品 / 满数量赠它品 的独立促销类型分流
- 满数量赠相关赠品弹窗中增加 `选择商品`
- 所有促销活动新增/编辑页中，商品选择/赠品选择弹窗修正错误表头展示
- 商品选择弹窗统一查询区，去掉 `查询维度`

## 2. 目标

### 2.1 菜单与类型

- 在 `满数量赠本品` 下方新增同级菜单 `满数量赠它品`
- `满数量赠本品`、`满数量赠它品` 作为两个独立促销类型存在
- 同时兼容历史 `满数量赠` 数据，避免旧活动、新建入口、筛选列表断裂

### 2.2 商品范围规则

#### 满数量赠本品

- 赠送方式默认仅展示 `本品`
- 不可编辑
- 不展示 `它品`

#### 满数量赠它品

- 赠送方式默认仅展示 `它品`
- 不可编辑
- 不展示 `本品`

#### 购买范围 / 满赠方式规则

- 单组 + 倍数：参与范围允许添加多行
- 多组 + 倍数：保持现状
- 单组 + 阶梯：保持现状
- 多组 + 阶梯：保持现状

这里的“保持现状”按用户补充口径处理：

- 参考当前 `满数量赠本品` 的交互规则
- 不扩散修改到其他促销类型

### 2.3 赠品弹窗联动

- 在 `满数量赠本品`、`满数量赠它品` 的新增/编辑页中
- 商品范围下的赠品选择弹窗增加按钮 `选择商品`
- 点击后展示选择商品弹窗
- 选择商品弹窗内容、样式、交互复用现有其他商品选择弹窗
- 选中的商品回填为赠品数据

### 2.4 弹窗通用修正

- 所有促销活动新增、编辑页中的商品选择弹窗
- 所有促销活动新增、编辑页中的赠品选择弹窗
- 修复列表首列表头把 `<input type="checkbox"...>` 当普通文本展示的问题

### 2.5 商品选择弹窗查询区统一

- 适用范围：所有促销活动新增、编辑页中的商品选择弹窗
- 去掉 `查询维度`
- 统一展示四个查询条件：
  - 商品
  - 类别
  - 品牌
  - 供应商

## 3. 非目标

- 不调整促销模板模块
- 不调整满金额赠逻辑
- 不改动门店选择弹窗
- 不新增真实后端接口或文件导入解析
- 不处理本轮未提及的 `商品折扣 -> 承担规则空白` 历史问题

## 4. 现状分析

## 4.1 满数量赠识别

当前 `campaignIsQtyGiftTemplate(tpl)` 仍基于模板名中包含 `满数量赠` 来判断，因此：

- 适合作为 `满数量赠本品` 与 `满数量赠它品` 的共同基础能力
- 需要在 `promoType` 展示、菜单筛选、默认 giftWay 上新增更细分的口径

## 4.2 商品范围逻辑

当前 `满数量赠本品` 已通过条件分支实现：

- `giftWay` 默认固定为 `本品`
- UI 中只读展示本品
- 单组 + 倍数允许多行

因此 `满数量赠它品` 最稳妥的做法不是复制一套新页面，而是：

- 在同一套 `giftScope` 渲染逻辑中
- 按当前活动 `promoType` 分成：
  - 数量赠本品
  - 数量赠它品
  - 其他满赠

## 4.3 选择弹窗现状

当前项目内存在两套相关弹窗：

- 通用选择弹窗 `openCampSelectModal(...)`
- 商品选择弹窗 `openCampaignGoodsSelectModal(...)`
- 赠品选择弹窗 `openCampaignGiftSelectModal(...)`

已确认：

- 商品选择弹窗已经是四个查询条件模式
- 但部分商品范围里的“选择商品”仍走旧的 `openCampSelectModal(...)`
- 旧通用弹窗存在 `查询维度`
- 通用 `table(...)` 会对表头做转义，因此传入 `<input ...>` 时会显示成文本，导致截图中的错误

## 5. 方案选择

本次采用方案 A：

- `满数量赠本品`、`满数量赠它品` 作为两个独立促销类型并列存在
- 底层共享同一套满数量赠商品范围框架
- 通过 `promoType` 决定 giftWay 默认值、只读展示文案、允许的交互

### 选择原因

- 列表筛选更清晰
- 新建入口更清晰
- 编辑回填更稳定
- 后续继续扩充本品/它品差异时不需要再返工

## 6. 详细设计

### 6.1 促销类型兼容策略

新增三个概念：

- 历史值：`满数量赠`
- 新值 1：`满数量赠本品`
- 新值 2：`满数量赠它品`

兼容口径：

- 历史活动如果是 `满数量赠`，在列表和入口上按 `满数量赠本品` 兼容展示
- 菜单筛选时：
  - `满数量赠本品` 匹配 `满数量赠本品` 和历史 `满数量赠`
  - `满数量赠它品` 只匹配 `满数量赠它品`
- 模板识别仍使用 `campaignIsQtyGiftTemplate(tpl)` 作为基础
- 新建/复制/编辑时，`promoType` 保持具体值，不再只回写成泛化的 `满数量赠`

### 6.2 菜单设计

`促销活动 -> 满减满赠` 子菜单顺序调整为：

1. 满金额减
2. 满数量减
3. 满金额赠
4. 满数量赠本品
5. 满数量赠它品
6. 满金额换购
7. 满数量换购

### 6.3 满数量赠本品 / 它品 的 giftWay 规则

新增两个辅助判断口径：

- 是否为数量赠本品活动
- 是否为数量赠它品活动

行为规则：

- 数量赠本品：
  - `giftWay = 本品`
  - UI 显示只读输入框：本品
- 数量赠它品：
  - `giftWay = 非本品`
  - UI 显示只读输入框：它品
- 其他满赠活动：
  - 保持当前本品 / 非本品单选逻辑

### 6.4 单组 + 倍数 多行规则

适用范围扩展为：

- 满数量赠本品
- 满数量赠它品

处理方式：

- 放开 `campGiftBuyAdd`
- 放开“选择商品批量加入单组参与范围”时的一条限制
- 只在 `购买范围=单组 && 满赠方式=倍数 && 数量赠本品/它品` 时生效

其余组合保持现状。

### 6.5 赠品弹窗增加“选择商品”

当前赠品弹窗仅支持在弹窗内勾选赠品行。

本次改为：

- 在赠品弹窗顶部操作区新增按钮 `选择商品`
- 点击后打开 `openCampaignGoodsSelectModal(...)`
- 从商品弹窗选中的商品，按赠品结构映射回当前赠品弹窗结果集

映射字段：

- `sku -> giftCode`
- `barcode -> giftBarcode`
- `name -> giftName`
- `originPrice -> giftPrice`
- `giftQty` 默认 1

该能力覆盖：

- 单组参与范围中的选择赠品
- 多组参与范围中的选择赠品
- 阶梯赠品范围中的选择赠品

### 6.6 弹窗错误表头修复

问题根因：

- 当前表格头部通过 `table(headers, rowsHtml)` 渲染
- `table()` 会转义表头字符串
- 因而 `<input type="checkbox"...>` 显示成普通文本

修正方式：

- 不再把复选框 HTML 直接塞入需要转义的通用表头字符串
- 为相关弹窗单独输出 `thead` HTML，或让表格方法支持“原始表头单元格”

本次优先采用最小改动方案：

- 在商品选择弹窗、赠品选择弹窗、通用选择弹窗这三处弹窗内部单独拼接表格
- 首列表头复选框按原始 HTML 输出
- 不扩大修改到全局所有列表页

### 6.7 商品选择弹窗统一查询区

统一目标：

- 不再使用 `查询维度`
- 固定展示以下条件：
  - 商品
  - 类别
  - 品牌
  - 供应商

处理口径：

- 所有促销活动新增/编辑页里的“选择商品”均优先改走 `openCampaignGoodsSelectModal(...)`
- 旧的 `openCampSelectModal(...)` 保留给类别、品牌等非商品弹窗
- 对仍然使用通用弹窗选择商品的入口切换到 `openCampaignGoodsSelectModal(...)`

## 8. 验收要点

### 8.1 菜单与列表

- 可见 `满数量赠本品`、`满数量赠它品` 两个同级菜单
- 点击 `满数量赠本品` 可看到本品活动及历史数量赠活动
- 点击 `满数量赠它品` 仅看到它品活动

### 8.2 商品范围

- 新增/编辑 `满数量赠本品` 时，赠送方式仅显示 `本品`
- 新增/编辑 `满数量赠它品` 时，赠送方式仅显示 `它品`
- 两者都不可编辑

### 8.3 单组 + 倍数

- `满数量赠本品` 单组 + 倍数可新增多行参与范围
- `满数量赠它品` 单组 + 倍数可新增多行参与范围

### 8.4 赠品弹窗

- 赠品弹窗可见 `选择商品` 按钮
- 点击后可打开商品选择弹窗
- 选中商品后能正确回填为赠品

### 8.5 表头错误修复

- 商品选择弹窗、赠品选择弹窗首列不再显示 `<input type="checkbox"...>` 原始文本
- 复选框可正常显示和勾选

### 8.6 查询区

- 商品选择弹窗不再显示 `查询维度`
- 统一显示：商品、类别、品牌、供应商

## 9. 风险与控制

- 风险：历史 `满数量赠` 数据被错误归到 `满数量赠它品`
  - 控制：历史值仅兼容到 `满数量赠本品`

- 风险：多个商品选择入口仍残留旧通用弹窗
  - 控制：逐个排查促销活动新增/编辑页相关入口，统一切换到 `openCampaignGoodsSelectModal(...)`

- 风险：表头修复影响其他普通表格
  - 控制：仅限选择弹窗内单独修正，不改全局列表表格函数
