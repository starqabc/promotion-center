# 礼券主题：券包管理页面设计

## 1. 背景

本次只调整 `礼券主题` 模块，在其二级菜单下新增一个同级菜单 `券包管理`。

项目约束保持不变：

- 仅 HTML / CSS / JS
- 无依赖
- 支持 `file://` 直接打开
- 复用现有左侧菜单、列表页、通用弹窗、表格与表单样式

用户需求：

1. 在礼券主题二级菜单下方添加同级菜单 `券包管理`
2. 页面内容参考所给截图
3. 截图中的字段文案里，`活动` 二字全部改为 `券包`

## 2. 目标

### 2.1 菜单

- 在 `礼券主题` 分组下新增同级菜单 `券包管理`
- 与 `礼券主题`、`礼券分类` 并列展示

### 2.2 页面形态

- `券包管理` 采用 **列表页 + 新增/编辑弹窗** 的组合模式
- 列表页负责查询、浏览、操作
- 新增/编辑通过通用弹窗承载长表单

### 2.3 文案口径

截图中涉及 `活动` 的字段统一改为 `券包`，例如：

- 活动ID -> 券包ID
- 活动名称 -> 券包名称
- 活动时间 -> 券包时间
- 活动周期 -> 券包周期
- 活动说明 -> 券包说明
- 活动图片 -> 券包图片
- 活动设置 -> 券包设置

## 3. 非目标

- 不改动现有 `礼券主题` 向导页
- 不改动 `礼券分类`
- 不新增真实上传能力
- 不接后端接口
- 不扩展审批、打印、统计等额外流程

## 4. 方案选择

采用方案 A：

- 新增一个独立菜单 `券包管理`
- 默认进入券包管理列表页
- 点击新增、修改、复制时打开长弹窗

### 选择原因

- 最接近用户提供的截图形态
- 能复用现有列表页与通用弹窗能力
- 相比新增独立向导页，改动范围更小

## 5. 页面设计

### 5.1 列表页

列表页结构参考项目中已有的礼券主题/礼券分类列表：

- 面包屑
- 查询区
- 查询操作按钮
- 列表操作按钮
- 数据表格

建议的查询条件：

- 券包ID
- 券包名称
- 状态
- 创建时间-起
- 创建时间-止

建议的列表操作：

- 新增
- 修改
- 复制
- 详情

### 5.2 弹窗结构

点击 `新增`、`修改`、`复制` 后打开通用 modal，大尺寸长弹窗，内容分为两块：

1. 基础设置
2. 券包设置

弹窗底部保留：

- 确定
- 取消

### 5.3 基础设置

字段参考截图，并将 `活动` 全部改为 `券包`：

- 券包ID
- 券包名称
- 券包时间
- 券包周期
- 会员等级
- 会员标签
- 发放份数
- 每人限领
- 券包说明
- 券包图片

说明：

- `券包ID` 在原型中可允许为空或只读展示
- `券包时间` 使用两个日期输入框
- `券包周期` 参考截图，采用单选项
- `券包图片` 采用静态上传占位框，不接真实上传

### 5.4 券包设置

字段参考截图：

- 领取方式
- 礼券售价
- 券明细表格

其中券明细表格包含：

- 删除
- 序号
- 礼券ID
- 礼券名称
- 券面值
- 赠送张数
- 券类型
- 有效期类型
- 有效开始日期/发券N天
- 有效终止日期/有效天数

表格底部提供：

- `点击这里追加记录...`

用于新增一行券明细。

## 6. 数据与交互设计

### 6.1 列表数据

为 `券包管理` 增加独立原型数据集，字段建议包含：

- packId
- packName
- status
- packStartDate
- packEndDate
- creator
- createdAt
- modifier
- modifiedAt
- approver
- approvedAt
- rows

### 6.2 弹窗草稿数据

新增临时草稿对象用于弹窗编辑，包含：

- 基础设置字段
- 券包设置字段
- 券明细数组

### 6.3 交互规则

- 新增：打开空白弹窗
- 修改：打开当前数据回填弹窗
- 复制：复制当前券包并生成新的券包ID
- 详情：可复用抽屉或只读弹窗展示基础信息和券明细
- 保存：仅更新前端原型数据并提示成功

## 7. 影响范围

核心文件：

- `index.html`
- `app.js`

可能涉及：

- 左侧礼券菜单
- 路由表
- 券包管理列表页渲染
- 券包管理弹窗渲染
- 列表操作事件
- 原型数据初始化

## 8. 验收要点

- 礼券主题菜单下新增 `券包管理`
- 点击后进入券包管理列表页
- 列表页可看到查询区、按钮区和表格
- 点击新增可打开长弹窗
- 弹窗字段中所有原本 `活动` 文案均已改为 `券包`
- 弹窗包含 `基础设置` 与 `券包设置`
- 券包设置中可新增券明细行

## 9. 风险与控制

- 风险：弹窗内容较长，视觉可能拥挤
  - 控制：复用通用大弹窗并保持分区标题

- 风险：列表与弹窗字段过多导致改动范围扩大
  - 控制：只做原型必需字段，不增加真实业务校验

- 风险：与现有礼券主题模块混淆
  - 控制：菜单、路由、数据集均独立命名为券包管理
