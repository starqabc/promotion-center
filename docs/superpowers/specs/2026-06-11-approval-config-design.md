## 背景

需要在系统管理下新增二级菜单“审批配置”，提供审批流程配置的列表页与流程设计器页。项目为纯前端原型（HTML/CSS/JS，无依赖，file:// 可打开），数据存于 AppState.data。

## 目标

- 系统管理下新增入口：审批配置
- 列表页：查询 + 列表 + 操作（查看/修改/发布/应用）+ 新增
- 设计器页：展示流程结构（开始/审批节点/分支/结束），支持选中节点并在右侧配置审批规则（处理方式、处理人、多 人审批方式、通过条件、处理人为空策略、任务转交人）
- 交互与样式参考截图，但实现为“可演示”的轻量版本

## 非目标

- 不实现真实的 BPMN/流程引擎
- 不实现后端持久化与权限
- 不实现复杂条件表达式、动态变量、跨节点校验

## 信息架构

- /sys-approval：审批配置列表
- /sys-approval-edit：审批配置设计器（新增/编辑）

## 数据模型（原型）

- sysApprovalFlows：[{ id, name, creator, lastPublishAt, publishStatus, status, version, graph }]
- graph（简化）：{ nodes: [{id,type,name,config}], links: [] }，并支持并行分支节点（type=parallel，包含 branches）
- sysUsers：[{ id, name, account, position, dept }]

## 列表页设计

### 查询条件

- 流程配置ID、流程名称、状态（全部/启用/停用）、发布状态（全部/已发布/未发布）

### 列表字段

- 序号、流程配置ID、流程名称、创建人、最后发布时间、发布状态、状态、版本号、操作

### 操作

- 新增：进入设计器（新增）
- 查看：抽屉查看基本信息 + 进入设计器按钮
- 修改：进入设计器（编辑）
- 发布：二次确认后更新发布状态与最后发布时间
- 应用：二次确认（原型 toast）

## 设计器页设计

### 布局

- 左侧：流程画布（纵向主干 + 可选并行分支展示）
- 右侧：节点配置面板（选中节点后展示）

### 节点配置（审批节点）

- 处理方式：人工处理 / 自动拒绝 / 自动跳过
- 处理人：岗位 / 指定人员 / 上级领导 / 部门负责人 / 连续多级上级审批 / 发起人
- 多人处理时审批方式：或签 / 会签
- 通过条件：全部同意 / 同意人数超过 N%
- 处理人为为空：前一个节点先走完 / 自动跳过 / 指定用户处理 / 当前节点挂起不处理
- 任务转交人：指定人员
- 选择人员：弹窗（查询方式 + 搜索 + 已选择）

### 保存与发布

- 保存：写入 AppState.data.sysApprovalFlows
- 发布：二次确认后将发布状态置为已发布并更新最后发布时间、版本号（+1）

## 风险与约束

- file:// 缓存：需要更新资源版本号
- 样式兼容：尽量复用现有 card/toolbar/table/tabbar/modal/drawer 组件
