# 促销活动：满立减菜单与商品范围裁剪设计

## 1. 背景

本次只调整 `促销活动` 模块中的满减促销类型，不扩展到模板、门店、其他促销玩法。

当前项目约束保持不变：

- 仅 HTML / CSS / JS
- 无依赖
- 支持 `file://` 直接打开
- 复用现有左侧菜单、Hash 路由、列表、向导页、弹窗、表格模式

用户新增需求：

1. 在 `满金额减` 下方创建同级菜单 `满立减`
2. `满立减` 页面所有内容参考 `满金额减`
3. 其中商品范围：
   - 不展示排除范围
   - 参与范围内不展示类别、品牌

用户已确认采用：

- `满立减` 作为**独立促销类型**

## 2. 目标

### 2.1 菜单与促销类型

- 在 `促销活动 -> 满减满赠` 分组中
- 于 `满金额减` 下方新增同级菜单 `满立减`
- `满立减` 在活动列表筛选、新增入口、编辑回填中作为独立促销类型存在

### 2.2 页面复用

- `满立减` 的新增/编辑页整体参考 `满金额减`
- 条件范围、承担规则、其他基础信息交互保持与 `满金额减` 一致

### 2.3 商品范围裁剪

仅对 `满立减` 的商品范围做差异化处理：

- 不展示排除范围
- 参与范围仅保留 `商品`
- 不展示 `类别`
- 不展示 `品牌`

## 3. 非目标

- 不调整 `满金额减` 原有逻辑
- 不改促销模板模块
- 不改满数量减、满金额赠、满数量赠等其他类型
- 不新增后端接口或真实数据结构
- 不额外调整商品选择弹窗以外的其他弹窗

## 4. 现状分析

### 4.1 满金额减识别

当前活动页对 `满金额减` 的识别依赖：

- `campaignIsAmountReduceTemplate(tpl)`
- `campaignIsQtyReduceTemplate(tpl)`

并通过这些判断决定：

- 条件范围是否显示
- 步骤数是否为 6 步
- 商品范围渲染逻辑是否走满减结构

### 4.2 商品范围实现

当前 `满金额减` 商品范围复用通用商品范围页：

- 参与范围包含 `商品 / 类别 / 品牌`
- 排除范围可见
- 通过对应的 tab 和表格渲染函数切换

因此本次 `满立减` 不适合复制一套新页面，而应在现有满减商品范围中加入一个“满立减场景”的裁剪分支。

## 5. 方案选择

采用方案 A：

- `满立减` 为独立促销类型
- 页面主体复用 `满金额减`
- 商品范围按 `满立减` 场景单独裁剪

### 选择原因

- 列表与菜单语义清晰
- 新建和编辑更稳定
- 后续如果 `满立减` 继续新增规则，可在当前独立促销类型上增量扩展

## 6. 详细设计

### 6.1 菜单设计

`促销活动 -> 满减满赠` 菜单顺序调整为：

1. 满金额减
2. 满立减
3. 满数量减
4. 满金额赠
5. 满数量赠本品
6. 满数量赠它品
7. 满金额换购
8. 满数量换购

### 6.2 促销类型口径

`满立减` 为独立促销类型，意味着：

- 左侧菜单路由使用 `满立减`
- 活动列表筛选可筛出 `满立减`
- 新增 `满立减` 时草稿 `promoType = 满立减`
- 编辑 `满立减` 时保留该值

同时，为了复用 `满金额减` 的新增/编辑结构，需要补一层“满减同源识别”：

- `满金额减`
- `满立减`

这两个类型在页面结构上按同一类处理，但在文案和筛选上独立。

### 6.3 模板匹配策略

本次不新增一套独立模板数据。

因此 `满立减` 的模板匹配策略为：

- 新增/编辑 `满立减` 时，沿用当前 `满金额减` 模板能力
- 模板识别仍按 `campaignIsAmountReduceTemplate(tpl)` 作为基础
- `promoType` 展示与保存时保持 `满立减`

即：

- 模板层复用 `满金额减`
- 活动层区分 `满立减`

### 6.4 步骤与条件范围

`满立减` 的新增/编辑页步骤与 `满金额减` 保持一致：

1. 基础信息
2. 用户范围
3. 门店范围
4. 条件范围
5. 商品范围
6. 承担规则

条件范围全部沿用 `满金额减`：

- 活动互斥
- 满减方式
- 折扣类型/减额规则
- 倍数/阶梯

本轮不额外新增 `满立减` 的条件差异。

### 6.5 商品范围裁剪

#### 6.5.1 排除范围

`满立减` 商品范围中不展示排除范围，因此：

- 不渲染排除范围 tab
- 不渲染排除范围表格容器
- 不展示排除范围相关按钮：
  - 新增一行
  - 导入
  - 选择

#### 6.5.2 参与范围

`满立减` 参与范围只保留 `商品`，因此：

- 不渲染 `类别` tab
- 不渲染 `品牌` tab
- 只保留 `商品` tab

对应地：

- 商品范围下的选择商品、导入、查询仍保留
- 类别、品牌相关入口在 `满立减` 页面不出现

### 6.6 数据结构策略

为了最小改动，`满立减` 仍使用当前 `goodsScope` 数据结构：

- `goods`
- `categories`
- `brands`
- `excludeGoods`
- `excludeCategories`

但在 `满立减` 页面中：

- `categories`
- `brands`
- `exclude*`

都不展示，也不作为用户交互入口暴露。

这样可以避免重构大量通用渲染函数。

### 6.7 列表与新建兼容

列表与新建的目标行为：

- 点击 `满立减` 菜单，进入 `满立减` 列表
- 点击列表新增，新增页默认 `promoType = 满立减`
- 若未选模板，仍能按 `满金额减` 模板能力带出页面结构
- 编辑已创建的 `满立减` 活动时，页面保持 `满立减` 身份

### 8.2 列表与新增

- 点击新增后进入 `满立减` 页面

### 8.3 页面主体

- `满立减` 的基础信息、用户范围、门店范围、条件范围、承担规则均参考 `满金额减`

### 8.4 商品范围

- 参与范围仅显示 `商品`
- 不显示 `类别`
- 不显示 `品牌`

## 9. 风险与控制

- 风险：`满立减` 因无独立模板而无法进入对应结构
  - 控制：模板匹配层复用 `满金额减` 识别

- 风险：商品范围裁剪误伤 `满金额减`
  - 控制：仅对 `promoType = 满立减` 的场景加条件分支

- 风险：编辑回填时 `promoType` 被模板重新覆盖成 `满金额减`
  - 控制：在模板回填、保存时保持 `满立减` 优先
