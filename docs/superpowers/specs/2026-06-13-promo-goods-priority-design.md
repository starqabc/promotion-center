# 促销商品优先级（原型）设计

## 背景
在“促销规则设置”下新增同级菜单“促销商品优先级”，用于维护“同一促销类型下，活动中商品的优先级顺序”。

## 目标
- 提供一个可直接参考的原型页面，交互与“模版规则设置”同风格：筛选 → 分组展开 → 上移/下移调整顺序 → 新增/编辑维护。
- 保持纯前端原型模式：无依赖、可 file:// 打开、数据只在内存里模拟。

## 信息结构
- 菜单位置：促销模版模块下，位于“模版规则设置”下方
- 路由：
  - `#/promo-goods-priority`：列表/分组排序
  - `#/promo-goods-priority-create`：批量新增
  - `#/promo-goods-priority-edit`：单条编辑

## 列表页（促销商品优先级）
- 筛选：
  - 促销类型（下拉）
  - 商品（编码/条码/名称模糊）
  - 状态（启用/停用）
- 分组展示：
  - 按“促销类型”分组
  - 行可展开，展示该类型下商品优先级明细
- 明细操作：
  - 上移/下移：在当前促销类型内重新排序，并自动重排优先级为 1..N
  - 编辑：进入编辑页

## 新增页（批量）
- 选择促销类型、状态
- “选择商品”弹窗批量添加
- 表格维护每个商品的优先级、支持删除行
- 保存后生成多条记录并返回列表

## 编辑页（单条）
- 展示规则编号、商品信息（只读）
- 允许修改促销类型、优先级、状态、说明

## 数据模型（原型）
每条记录结构：
- `id`：`PGP-YYYY-####`
- `promoType`：促销类型
- `goodsCode / goodsBarcode / goodsName / goodsPrice`
- `priority`：数字，越小越优先（同类型内 1..N）
- `status`：启用/停用
- `remark`：说明
- `creator / createAt / modifier / modifyAt`
