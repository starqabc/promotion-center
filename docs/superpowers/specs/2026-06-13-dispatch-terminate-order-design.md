# 促销调度终止单原型设计

## 1. 背景

本次只调整 `促销调度` 一级菜单下的原型页面，不修改其他业务模块。

项目约束保持不变：

- 仅 HTML / CSS / JS
- 无依赖
- 支持 `file://` 直接打开
- 左侧菜单 + Hash 路由
- 复用现有列表页、整页表单、详情页原型模式

用户本轮需求：

1. `生效商品` 列表页去掉 `批量终止` 按钮
2. `生效商品` 列表去掉所有复选框
3. `生效商品` 列表去掉操作列
4. 在 `促销调度` 一级菜单下新增二级菜单 `促销终止单`
5. `促销终止单` 需要包含：
   - 列表页
   - 新增页
   - 详情页
6. 其他不做处理

用户已确认本轮采用：

- 独立新增 `促销终止单` 的列表 / 新增 / 详情
- 仅精简 `生效商品` 列表

## 2. 目标

### 2.1 精简 `生效商品`

保留 `生效商品` 的查询区和只读列表浏览能力，但去掉终止操作能力。

调整后：

- 不显示 `批量终止` 按钮
- 不显示首列复选框
- 不显示操作列

### 2.2 新增 `促销终止单`

在 `促销调度` 下新增独立二级菜单 `促销终止单`，并提供：

1. 列表页
2. 新增页
3. 详情页

## 3. 非目标

- 不修改真实促销活动数据结构的终止逻辑
- 不将 `生效商品` 改造成可新增终止单页面
- 不新增审批流、打印、导出等完整业务流程
- 不扩展到其他一级菜单或促销活动页面

## 4. 现状分析

### 4.1 现有 `促销调度`

当前 `index.html` 中，`促销调度` 下只有一个二级菜单：

- `生效商品`

### 4.2 现有 `生效商品`

当前 `app.js` 中的 `renderPromoGoodsTerminatePage()` 同时承担：

- 查询生效商品
- 勾选商品
- 单条终止
- 批量终止
- 查看促销活动详情

因此本次需要将其收口为纯浏览列表，同时把“终止单”能力独立为新的页面组。

## 5. 方案选择

本次采用方案 A：独立新增 `促销终止单` 路由组。

### 5.1 备选方案

- 方案 A：独立路由组
- 方案 B：合并到 `生效商品`
- 方案 C：列表页 + 抽屉式新增/详情

### 5.2 选择原因

- 与用户确认的范围一致
- 页面职责更清晰
- 最符合截图中的整页新增 / 详情布局
- 对现有 `生效商品` 改动最小

## 6. 详细设计

### 6.1 菜单结构

`促销调度` 下调整为：

1. `生效商品`
2. `促销终止单`

其中：

- `生效商品` 路由保持 `/promo-goods-terminate`
- `促销终止单` 新增独立路由

建议路由：

- `/promo-terminate-orders`
- `/promo-terminate-orders-create`
- `/promo-terminate-orders-detail/<id>`

## 6.2 `生效商品` 列表页调整

保留：

- 查询区
- 列表标题
- 当前商品数据源

删除：

- `批量终止` 按钮
- 列表首列复选框
- `操作` 列
- 与 `pgtTerminateBatch / pgtTerminate / pgtDetail` 相关的页面入口展示

说明：

- 本轮只从 UI 侧移除这些入口
- 内部历史辅助函数可视情况保留，不强制整轮清空

### 6.3 `促销终止单` 列表页

列表页参考首张截图，展示一张终止单主列表。

#### 查询区

建议字段：

- 促销终止单号
- 促销单号
- 商品
- 终止方式
- 终止日期
- 单据状态
- 生效状态
- 制单人
- 制单日期
- 审核日期

#### 按钮区

保留原型按钮：

- 新增
- 重置
- 查询

可做静态展示按钮：

- 导出
- 打印

#### 列表字段

- 序号
- 终止单号
- 终止方式
- 终止时间
- 单据状态
- 生效状态
- 制单人
- 制单时间
- 审核人
- 审核时间
- 备注
- 操作

操作列仅保留：

- 查看

### 6.4 `新增促销终止单` 页面

采用整页表单布局，按截图拆为两个区域：

#### 区块一：基础信息

字段：

- 终止方式
  - 按商品
  - 按类别
  - 按品牌
  - 按单据
  - 按门店
- 终止时间
- 备注

按不同终止方式，展示对应补充字段：

- 按商品：展示“终止门店”，并展示商品信息表格区
- 按类别：展示“终止门店” + “终止类别”
- 按品牌：展示“终止门店” + “终止品牌”
- 按单据：仅保留基础字段与下方促销信息查询区
- 按门店：展示“终止门店”

本轮实现以原型交互为准，不做复杂联动校验，只做必要的显示切换。

#### 区块二：促销信息

查询字段：

- 促销单号
- 促销类型
- 促销状态
- 制单人
- 制单日期

按钮：

- 重置
- 查询

列表区：

- 默认空态提示
- 用于承接按条件筛选出的促销活动列表

#### 页面动作

- 保存
- 返回列表

### 6.5 `促销终止单详情` 页面

详情页复用新增页布局，采用查看态：

- 标题显示终止单号
- 基础信息只读展示
- 促销信息结果区只读展示
- 不提供编辑保存

### 6.6 数据与状态设计

建议新增以下状态：

- `AppState.ui.promoTerminateOrders`
  - 存放列表页查询条件
- `AppState.ui.promoTerminateOrderForm`
  - 存放新增页临时表单条件
- `AppState.data.promoTerminateOrders`
  - 存放终止单原型数据

建议提供少量原型数据，覆盖：

- 按单据
- 按商品
- 按类别
- 按品牌

以支持列表页和详情页展示。

### 6.7 路由与导航高亮

需要同步调整：

- `index.html`：新增 `促销终止单` 二级菜单
- `app.js.routes`：新增 3 个路由标题
- `setActiveNav(r)`：让 `/promo-terminate-orders*` 高亮到新菜单
- `navSectionFromRoute(r)`：让新路由仍归属 `dispatch`
- 页面分发逻辑：接入列表 / 新增 / 详情渲染

## 7. 影响文件

### 7.1 `index.html`

负责：

- 在 `促销调度` 下新增 `促销终止单` 二级菜单

### 7.2 `app.js`

负责：

- 精简 `生效商品` 列表
- 新增终止单路由标题
- 新增终止单列表 / 新增 / 详情渲染函数
- 新增终止单页面事件处理与临时状态
- 更新导航高亮与分组归属

### 7.3 `styles.css`

仅在新增页表单或列表布局与现有样式不匹配时补少量样式，不做大范围改造。

## 8. 验收标准

- `生效商品` 不再显示 `批量终止` 按钮
- `生效商品` 不再显示复选框列
- `生效商品` 不再显示操作列
- `促销调度` 下新增 `促销终止单`
- `促销终止单` 列表页可查看样例数据
- `促销终止单` 列表页可进入新增页
- `促销终止单` 列表页可进入详情页
- 新增页可按终止方式切换展示基础信息字段
- `index.html`、`app.js`、必要时的 `styles.css` 无新增 diagnostics
