## 背景

需要对 3 个既有模块做原型调整，且只修改本次提到的页面与交互，不扩展到其他菜单。项目为纯前端原型（HTML/CSS/JS，无依赖，file:// 可打开）。

## 目标

- 模版优先级列表去掉“新增”按钮，改为按模版类型分组的主子表
- 档期智能促销新增/编辑页第 3 步门店范围去掉门店信息卡片，改成 tab + 列表
- 促销活动列表操作列新增“审批”，点击后展示审批弹窗

## 非目标

- 不重构模版优先级新增/编辑页
- 不修改档期智能促销其它步骤
- 不改促销活动详情页、创建页、审批配置页

## 方案一

### 模版优先级

- 列表页去掉“新增”按钮
- 列表按“模版类型”分组
- 主表字段：
  - 展开符
  - 序号
  - 模版类型
- 子表字段：
  - 序号
  - 规则编号
  - 模版编号
  - 模板名称
  - 优先级
  - 修改人
  - 修改时间
  - 操作
- 子表操作仅保留：
  - 上移
  - 下移
- 主表默认收起，点击 `+` 展开，点击 `-` 收起
- 上移/下移只在当前模版类型分组内生效，并同步重排优先级顺序

### 档期智能促销

- 只调整新增/编辑页第 3 步“门店范围”
- 去掉当前按分组包裹的门店信息卡片
- 保留 `门店名称1 / 门店名称2` tab
- 点击某个 tab 后，展示该 tab 对应的门店列表
- 列表字段：
  - 序号
  - 门店编码
  - 门店名称
  - 区域
  - 价格组
  - 备注

### 促销活动

- 所有促销活动列表操作列新增“审批”
- 点击“审批”后展示审批弹窗
- 弹窗字段：
  - 审批结果：审批通过 / 审批驳回（单选）
  - 审批说明：多行文本
- 原型行为：
  - 审批通过：活动状态更新为“审核”
  - 审批驳回：活动状态更新为“输入”
  - 同步更新审核人、审核时间、修改人、修改时间

## 数据与状态

### UI 状态

- `AppState.ui.templatePriority.expandedTypes`
  - 记录模版优先级列表当前展开的模版类型

### 既有数据复用

- 模版优先级继续使用 `AppState.data.templatePriorities`
- 档期智能促销继续使用 `storeScope.groups`
- 促销活动继续使用 `AppState.data.campaigns`

## 渲染调整

- `renderTemplatePriorityPage`
  - 输出主子表分组结构
  - 去掉新增按钮
- 档期智能促销新增/编辑页
  - 第 3 步门店范围改为单一列表区域
- `renderCampaignsListPage`
  - 操作列加入“审批”

## 交互与事件

- 模版优先级：
  - 展开/收起分组
  - 分组内上移/下移
- 档期智能促销：
  - 复用现有 tab 切换状态，只改展示容器
- 促销活动：
  - 新增 `campApprove`
  - 审批弹窗保存后更新活动状态与审批信息

## 风险与约束

- 模版优先级列表改为分组后，需要保证查询条件仍然生效
- 档期智能促销编辑页需要兼容现有门店分组数据为空时的空态展示
- 审批弹窗不能影响原有“提交审核/生效/作废”等操作链路
