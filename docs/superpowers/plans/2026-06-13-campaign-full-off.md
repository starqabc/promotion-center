# 满立减菜单与商品范围裁剪实施计划

> **目标：** 在促销活动中新增独立促销类型 `满立减`，页面主体复用 `满金额减`，并在商品范围中隐藏排除范围、仅保留商品参与范围。
> **架构：** 基于现有 `app.js` 中的活动子菜单、促销类型兼容、满减步骤渲染和商品范围渲染逻辑做增量修改。通过新增 `满立减` 类型判断、复用 `满金额减` 结构、在商品范围内按 `满立减` 场景裁剪 tab 和容器来实现，不复制新页面。

---

- `app.js`

### 任务 1：补齐满立减促销类型与菜单

**涉及文件：**
- 修改：`app.js`

- [ ] **步骤 1：补充促销类型判断**
  - 新增 `满立减` 的独立促销类型判断
  - 同时补一层“满减同源类型”判断，用于复用 `满金额减` 页面结构

- [ ] **步骤 2：更新子菜单顺序**
  - 在 `满减满赠` 分组中，将 `满立减` 插入到 `满金额减` 下方

- [ ] **步骤 3：更新列表筛选与新建带入**
  - 点击 `满立减` 菜单时，列表筛选只筛 `满立减`
  - 从 `满立减` 入口新建时，草稿默认 `promoType = 满立减`

- [ ] **步骤 4：更新编辑与保存回填**
  - 编辑 `满立减` 活动时，保持 `promoType = 满立减`
  - 保存时不要被模板识别重新覆盖成 `满金额减`

- [ ] **步骤 5：运行诊断**
  - 使用 IDE diagnostics 检查 `app.js`
  - 预期：无新增 diagnostics

- [ ] **步骤 6：手动验收**
  - 刷新页面
  - 检查菜单是否新增 `满立减`
  - 检查点击 `满立减` 时列表筛选是否正常

---

### 任务 2：让满立减复用满金额减页面结构

**涉及文件：**
- 修改：`app.js`

- [ ] **步骤 1：复用步骤判断**
  - 让 `满立减` 走与 `满金额减` 相同的步骤数
  - 保持：
    - 基础信息
    - 用户范围
    - 门店范围
    - 条件范围
    - 商品范围
    - 承担规则

- [ ] **步骤 2：复用条件范围逻辑**
  - 让 `满立减` 沿用当前 `满金额减` 的条件范围渲染和数据结构

- [ ] **步骤 3：复用承担规则逻辑**
  - 不新增 `满立减` 专属承担规则
  - 直接沿用 `满金额减`

- [ ] **步骤 4：运行诊断**
  - 使用 IDE diagnostics 检查 `app.js`
  - 预期：无新增 diagnostics

- [ ] **步骤 5：手动验收**
  - 从 `满立减` 入口进入新增页
  - 检查步骤、条件范围、承担规则是否与 `满金额减` 一致

---

### 任务 3：裁剪满立减商品范围

**涉及文件：**
- 修改：`app.js`

- [ ] **步骤 1：限制参与范围 tab**
  - `满立减` 商品范围中只保留 `商品`
  - 不展示 `类别`
  - 不展示 `品牌`

- [ ] **步骤 2：隐藏排除范围**
  - 在 `满立减` 商品范围中不渲染排除范围 tab
  - 不渲染排除范围容器与相关按钮

- [ ] **步骤 3：保留商品范围基础操作**
  - 保留商品维度下的：
    - 查询
    - 重置
    - 新增一行
    - 导入
    - 选择商品

- [ ] **步骤 4：避免误伤满金额减**
  - 确保以上裁剪只在 `promoType = 满立减` 时生效
  - `满金额减` 的商品/类别/品牌与排除范围保持原样

- [ ] **步骤 5：运行诊断**
  - 使用 IDE diagnostics 检查 `app.js`
  - 预期：无新增 diagnostics

- [ ] **步骤 6：手动验收**
  - 打开 `满立减` 新增/编辑页
  - 检查商品范围只剩商品参与范围
  - 检查不再显示排除范围
  - 再打开 `满金额减` 页面确认原行为未变
